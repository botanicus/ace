After a long time I decided to blog again. Let me shortly introduce myself:

I'm a Ruby developer currently living in London and I'm a senior developer at awesome company Headshift. My nickname is @botanicus and I'm interested in Ruby, Django, JavaScript, Node.js, Erlang, Riak & similar stuff.

I was working with great Merb framework for few years till Engine Yard fucked it up and decided to develop shitty Rails.

I have more than few projects at GitHub, the most important are:

  • Rango is my light-weight web framework with template inheritance and much more interesting features. Currently I'm working on version 0.3 which will introduce apps/plugins distributions as gems, which will basically mean that all the apps will be reusable as plugins in other apps.
  • Pupu is a package manager for stuff living in public/ like icon sets, JS/CSS framework include plugins like various lightbox implementations, table sorting etc. It can find & download plugins from GitHub and install them, include their dependencies. It also provides helpers for including these plugins with dependencies to your templates etc.
  • Nake is my Rake replacement with advanced CLI parsing, configuration for tasks and much more.
  • Git-Deployer is my simple deployment solution based on Git post-receive hook include deployment to multiple servers. It's framework independent, you can use it for Django, Rango, Rails, whatever.

Well, enough about me. Regarding the topics I'm going to write about, don't expect any PR bollocks like “oh cool, the new jQuery is so awesome” (it's not BTW). I want to cover topics like “why your favourite framework is a crap”, write more about patterns and approaches in IT etc.
